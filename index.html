Chaos Engineering

Using Litmus Chaos experiments, I created a consumable product for microservices migrating or developing on AWS to run chaos tests to ensure the resiliency of their service. Helm charts were also used to streamline running chaos tests by limiting the amount of information needed to run tests. Additionally, I learned and implemented Argo workflows to create chaos workflows to allow for service owners to run a series of chaos tests that will prove resiliency.

MapMyHerd

Currently, in the agriculture field, there is no software to manage and monitor cattle location and activity data for cattle ranches. While there are tools for managing animal records, they are cumbersome to use. MapMyHerd provides actionable information to enable ranchers to prioritize their most precious resource–their time–while improving operational efficiency. Data-collecting sensors on a cattle ear tag provides information to the comprehensive web app “herd map” with an intuitive, easy-to-understand design and user flow that provides a one-stop-shop for a rancher to see where all their cattle are and how they are doing - quite literally “where’s the beef?” 

  • Client: Roper Solutions, Inc.
  • Date: August 2021 - May 2022
  • Service: Web Development
